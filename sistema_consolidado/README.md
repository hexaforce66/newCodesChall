# 🚀 Reporte Consolidado

**OBJETIVO PRINCIPAL**
El objetivo principal de este programa es procesar transacciones bancarias, actualizando los saldos de las cuentas en una base de datos según las transacciones registradas en un archivo de texto.

**FLUJO FUNCIONAL**
1. El programa inicia leyendo un archivo de texto que contiene las transacciones bancarias.
2. Para cada transacción, el programa consulta el saldo actual de la cuenta en la base de datos y, si es válido, actualiza el saldo según la transacción.
3. Si la actualización es exitosa, el programa registra el éxito y continúa con la siguiente transacción.
4. Al finalizar el procesamiento, el programa muestra un resumen de las transacciones procesadas, incluyendo el total de transacciones leídas, procesadas con éxito y con errores.

**VALOR DE NEGOCIO**
Si este programa fallara, podría tener un impacto significativo en la operación del banco, ya que las transacciones no se procesarían correctamente y los saldos de las cuentas no se actualizarían. Esto podría generar problemas para los clientes, como retrasos en la disponibilidad de fondos o errores en sus estados de cuenta. Además, el banco podría enfrentar riesgos operativos y financieros si no se pueden procesar las transacciones de manera oportuna y precisa.

## 📋 2. Lógica
**REGLAS DE NEGOCIO**

1.  El monto de la transacción debe ser positivo.
2.  No se permite sobregiro (el saldo actual más el monto de la transacción debe ser mayor o igual a cero).
3.  La cuenta debe existir en la base de datos para realizar la transacción.
4.  El archivo de transacciones debe poder abrirse correctamente.
5.  La conexión a la base de datos debe establecerse correctamente.

**MATRIZ DE DECISIONES Y FÓRMULAS**

| Condición | Acción |
| --------- | ------ |
| Monto > 0 | Procesar transacción |
| Monto <= 0 | Rechazar transacción (monto inválido) |
| Saldo actual + Monto >= 0 | Permitir transacción (no sobregiro) |
| Saldo actual + Monto < 0 | Rechazar transacción (sobregiro) |
| Cuenta existe en DB | Procesar transacción |
| Cuenta no existe en DB | Rechazar transacción (cuenta no existe) |
| Archivo de transacciones abierto correctamente | Procesar transacciones |
| Archivo de transacciones no se puede abrir | Error crítico |
| Conexión a DB establecida correctamente | Procesar transacciones |
| Conexión a DB no se puede establecer | Error crítico |
