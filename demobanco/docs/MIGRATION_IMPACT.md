**Análisis de Impacto**

El programa DEMOBANCO es un sistema de transacciones bancarias que verifica si un monto de transacción excede el límite diario establecido. El impacto de este programa es moderado, ya que se utiliza para procesar transacciones financieras y tomar decisiones basadas en límites establecidos.

**Complejidad**

La complejidad del programa es baja, ya que se trata de un sistema de transacciones simples que utiliza estructuras de control condicionales (IF-ELSE) para tomar decisiones. El programa no utiliza estructuras de datos complejas ni algoritmos sofisticados.

**Plan de Fases**

A continuación, se presenta un plan de fases para implementar y mantener el programa DEMOBANCO:

**Fase 1: Análisis y Diseño**

* Identificar los requisitos del sistema y los límites diarios establecidos.
* Diseñar la estructura de datos y las estructuras de control condicionales.

**Fase 2: Implementación**

* Codificar el programa en COBOL utilizando la estructura de datos y las estructuras de control condicionales diseñadas.
* Realizar pruebas unitarias para asegurarse de que el programa funcione correctamente.

**Fase 3: Pruebas y Validación**

* Realizar pruebas de integración para asegurarse de que el programa funcione correctamente con diferentes escenarios de transacciones.
* Validar que el programa cumpla con los requisitos establecidos y los límites diarios.

**Fase 4: Implementación y Mantenimiento**

* Implementar el programa en un entorno de producción.
* Realizar mantenimiento regular para asegurarse de que el programa siga funcionando correctamente y se ajuste a los cambios en los requisitos y límites diarios.

**Fase 5: Monitoreo y Evaluación**

* Monitorear el rendimiento del programa y evaluar su eficacia en la toma de decisiones.
* Realizar ajustes y mejoras al programa según sea necesario.
